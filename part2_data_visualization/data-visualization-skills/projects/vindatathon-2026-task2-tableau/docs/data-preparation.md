# Data Preparation

## Prepared Output Contract

All prepared Tableau extracts are generated by `scripts/prepare_tableau_data.py`.

| Output | Grain | Purpose |
|---|---:|---|
| `orders_enriched.csv` | order | Customer, geography, payment, fulfillment, status, revenue, margin, promotion, return, and review context |
| `order_items_enriched.csv` | order item | Product/category/segment/size, promotion, line margin, return, review, and order context |
| `daily_kpi.csv` | day | Official sales revenue/COGS, orders, traffic, rolling trends, conversion proxy |
| `inventory_monthly.csv` | product-month | Stockout, overstock, fill rate, sell-through, risk quadrant, replenishment recommendation |
| `growth_forecast_monthly.csv` | month | Historical actuals plus six-month predictive EDA forecast from `sales.csv` only |
| `seasonality_indices.csv` | month/weekday | Peak/soft period indicators |
| `predictive_signals.csv` | signal/entity | Promotion, payment, return, inventory, and channel risk signals |
| `recommendations.csv` | action | CEO-facing prescriptive action list |
| `dashboard_kpis.csv` | KPI | Small KPI strip and metric definition table |

## Metric Rules

- Official revenue and COGS totals come from `sales.csv`.
- Product, promotion, customer, payment, return, and fulfillment slicing uses item/order-level joins.
- Revenue reconciles exactly between `sales.csv` and item-level estimated revenue in the current prepared output.
- Forecasts are EDA signals only; they are not Part 3 Kaggle predictions.

## Validation

See `data-preparation-validation.md`.

Key validation result:

- All tested primary keys are unique.
- All tested core foreign-key relationships have 100% coverage.
- `sales.csv` revenue equals item-level estimated revenue with 0 gap.
- Excluded files were not used.

## Null Policy

`orders_enriched.csv` and `order_items_enriched.csv` include sparse event fields by design. Review fields are blank for records without reviews, return fields are blank for records without returns, promotion metadata is blank for non-promoted line items, and shipment fields are blank for orders without shipment rows.

Fulfillment timing note: in the current prepared extracts, nonblank `fulfillment_days` ranges from 2 to 10 days. A Power BI bucket such as `11+ days` is therefore an unused fallback bucket, not evidence of missing data.

- `order-items-enriched-null-policy.md`
